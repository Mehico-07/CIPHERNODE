# Ciphernode

## Introduction

**Ciphernode** is a privacy-first communication platform built on the Briar/Bramble stack.  
It is designed for resilient, decentralized messaging where users can communicate without depending on a central server.

## Features

### Communication Features
- Direct peer-to-peer messaging with contact-based trust.
- Support for private conversations, forums, and asynchronous exchange workflows.
- Headless API support for automation and custom client integration.

### Mailbox Support
- Integrated mailbox capabilities for delayed/offline delivery workflows.
- Background mailbox sync and update handling for intermittent connectivity.
- Mailbox setup and management flows available in the Android client.

### Bluetooth and Nearby Connectivity
- Bluetooth-based local transport support for nearby communication scenarios.
- Alternative transport modules to improve reliability when internet access is limited.
- Local-first communication pathways to reduce dependency on traditional network infrastructure.

## Ciphernode vs Standard Chatting Apps

| Area | Standard Chat Apps | Ciphernode |
|---|---|---|
| Architecture | Usually centralized servers | Decentralized, peer-oriented model |
| Privacy Model | Provider-mediated metadata and routing | Privacy-first design with local control |
| Connectivity | Internet-first | Multi-transport approach (including local options) |
| Resilience | Platform/service dependent | Built for robust communication under constrained conditions |

## Crypto Techniques

Ciphernode inherits strong cryptographic building blocks from the underlying stack, including:
- Public-key cryptography for identity and trust establishment.
- Authenticated encryption for secure message confidentiality and integrity.
- Key agreement and transport key management for secure session evolution.
- Digital signatures for authenticity and tamper resistance.
